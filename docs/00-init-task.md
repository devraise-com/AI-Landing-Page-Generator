# AI Product Engineer - Test

## Goal

Build a simple AI Landing Page Generator

## Limitations

- Up to 6 hours total
- 100% of code should be created with LLMs (Claude Code / Codex / etc.)

## Task

- Build a small web app where a user enters a prompt describing a product or offer, and the system generates a landing page plan: page structure, section copy, and visual directions/examples for each section.
- The user can review and approve this draft, after which the app generates the final landing page.
- The final result should be mobile-first, easy to review, and presented as a real user-facing product, not just raw JSON or text.

## Scope

- Keep the pipeline simple but complete: prompt → draft landing page structure + visual ideas → user approval → final landing page.
- You do not need to build a full design tool or CMS.
- You can use one or more LLM calls, predefined UI components, simple mock visual generation/examples, and lightweight persistence if needed.
- The goal is to show that you can build a multi-step AI product with clear UX, not just call one model once.

## Requirements

- The app must be deployed and accessible via a public link.
- Code should be in a GitHub repository.
- In the README, briefly explain the architecture, how the generation pipeline works, how you used AI tools during development, and what you intentionally simplified to fit the 6-hour limit.

## Focus

- Prioritize end-to-end delivery, product quality, and pragmatic scope.
- We are evaluating whether you can use AI to build a small but real B2C product with multiple operations, a clean approval step, and a polished mobile-friendly final output.
- Do not overengineer.
